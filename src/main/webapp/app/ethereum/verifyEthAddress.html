How it works?

To verify identity of Ethereum address private key owner (a person who can send transactions from this address), you have to sign a simple message with yor verified OpenPGP key, and send this message from your Ethereum address to our smart contract. Then you request our system to pull this signed message from smart contract and check it's signature. If signature can be verified, our server puts user data (first name, last name, birthdate, nationality, OpenPGP key fingerprint, Ethereum address) to smart contract.

User data from this smart contract can be read by other smart contracts and decentralized applications. You can use your verified identity to make legal binding and also legally enforceable contracts/documents on the Ethereum blockchain.

Ethereum address can be connected to one OpenPGP key only, and one time only.

If OpenPGP key expires, user have to use another Ethereum address for new OpenPGP public key.

You can also read from and write to our smart contract from etherscan
For developers: this service created to be as much smart contract developer friendly as possible. See smart contract verified code on etherscan Please, ask questions in our telegram group , send issues to issue tracker
